# 软件开发环境搭建说明

本文档用于说明“北京交通大学就餐仿真系统”的开发环境搭建方法。

## 1. 软件开发工具下载安装

### 1.1 基础工具
1. **Git**：用于拉取与管理代码。
2. **JDK 21**：后端编译运行必需（`pom.xml` 指定 `java.version=21`）。
3. **Maven 3.x**：或直接使用项目提供的 `mvnw`。
4. **Node.js 18+（建议 LTS）**：用于前端依赖安装与运行。
5. **MySQL 8.0+**：用于业务数据存储。
6. **Docker Desktop / Docker Engine（可选但推荐）**：快速启动数据库。
7. **IDE（任选）**：IntelliJ IDEA / VS Code。

### 1.2 下载建议
- JDK：Oracle/OpenJDK 官方发行版（Temurin 等）。
- Node.js：官网 LTS 版本。
- Docker：官方安装包。
- IDE：官网安装包，按默认选项安装即可。

## 2. 依赖 SDK、插件与其他依赖安装

### 2.1 后端依赖
项目 `pom.xml` 已声明主要依赖，执行以下命令自动下载：

```bash
cd /home/runner/work/canteen/canteen
bash mvnw clean compile
```

主要依赖包括：Spring Boot、MyBatis、MySQL 驱动、JWT、Lombok 等。

### 2.2 前端依赖

```bash
cd /home/runner/work/canteen/canteen/canteen-frontend
npm install
```

将安装 Vue、Vite、Vue Router、Element Plus、ECharts、Axios、Pinia 等。

### 2.3 数据库初始化
方式一（推荐，Docker）：
```bash
cd /home/runner/work/canteen/canteen
docker-compose up -d
```

方式二（手工）：
1. 新建数据库 `canteen`；
2. 执行 `docs/db/schema.sql` 建表；
3. 执行 `docs/db/data_init.sql` 初始化数据（如课程环境需要）。

## 3. 配置说明（环境变量等）

### 3.1 JDK 与 Maven 环境变量（Windows 示例）
1. 配置 `JAVA_HOME` 指向 JDK 21 安装目录。
2. 将 `%JAVA_HOME%\bin` 加入 `Path`。
3. 若使用本地 Maven，配置 `MAVEN_HOME` 并加入 `Path`。

Linux/macOS 可在 shell 配置文件中添加：
```bash
export JAVA_HOME=/path/to/jdk-21
export PATH=$JAVA_HOME/bin:$PATH
```

### 3.2 数据库配置
后端默认配置见：`src/main/resources/application.yml`。
常用项：
- 数据库地址：`localhost:3306`
- 数据库名：`canteen`
- 用户名：`root`
- 密码：`1234`

如本机配置不同，请修改 `application.yml` 中 datasource 配置。

### 3.3 前端接口地址
前端请求配置在 `canteen-frontend/src/utils/request.js`。
如后端端口变更，请同步修改 baseURL。

## 4. HelloWorld 工程与环境验证

为确认开发环境搭建成功，建议做“最小可运行验证”：

### 4.1 后端 HelloWorld 验证（Spring Boot）
1. 启动后端：
```bash
cd /home/runner/work/canteen/canteen
bash mvnw spring-boot:run
```
2. 在浏览器或 curl 访问任一公开接口（例如食堂列表）：
```bash
curl http://localhost:8080/api/v1/user/canteens
```
3. 若返回 JSON（code/message/data），说明后端环境可用。

### 4.2 前端 HelloWorld 验证（Vite）
1. 启动前端：
```bash
cd /home/runner/work/canteen/canteen/canteen-frontend
npm run dev
```
2. 访问终端提示地址（通常为 `http://localhost:5173`）。
3. 页面可正常打开并可跳转到登录页/食堂页，说明前端环境可用。

### 4.3 联调验证
1. 后端与数据库均启动；
2. 前端可调用后端接口并正常展示数据；
3. 可执行 `tests/full_link_test.py` 做全链路回归（需先安装 Python requests）。
